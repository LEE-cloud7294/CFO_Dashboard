## 1. 프로젝트 정의

**한 줄 정의:** 단순 CFO 대시보드가 아닌, 회사의 방향성을 결정하는 경영 의사결정 지원 도구.
인건비 비중이 높아 줄여야 하는지, 원재료가 상승하는지 등 모든 방향성을 판단할 수 있는 툴.
CEO / CFO 최고관리자 단 1인 전용.

**세 가지 역할:**
1. 과거 진단 — 숫자가 어떻게 변했나 (추세)
2. 현재 신호 — 지금 어디가 문제인가 (경보)
3. 방향 근거 — 이 추세면 다음 달 어떻게 되나 (판단 근거)

**사용자:** 경영자 단 1인. 차변/대변 화면 노출 금지.

**입력:** 위하고 월별 분개장 CSV.
- 보유 데이터: 2022년(4~12월) / 2023 / 2024 / 2025 전체
- 매달 1개월치 추가 업로드로 누적
- 2022년은 공장 매매 이벤트 있는 해 (자산 급변 정상)

## 7. 데이터 정제 로직 (loader.py)

```python
def clean_journal(df):
    df = df[df['구분'].isin(['차변','대변'])].copy()
    df['전표일자'] = pd.to_datetime(df['전표일자'], errors='coerce')
    df = df[df['전표일자'].notna()]
    for col in ['차변','대변']:
        df[col] = pd.to_numeric(df[col].astype(str).str.replace(',',''), errors='coerce').fillna(0)
    df['계정그룹'] = df['계정코드'].astype(str).str[:1]
    return df
```

# ✅ 매출액 (Code 404 대변만 — 절대 400·455 포함 금지)
매출액 = df[df['계정코드']=='404']['대변'].sum()

# ✅ 원재료·부재료 (현금매입 기준 — 재고 개념 없음)
원재료매입 = df[df['계정코드']=='153']['차변'].sum()
매입할인   = df[df['계정코드']=='155']['대변'].sum()
원재료순   = 원재료매입 - 매입할인
부재료매입 = df[df['계정코드']=='162']['차변'].sum()

# ✅ 영업비용 (5xx+8xx, cost_bucket 기준)
# 원재료·부재료·일회성손익 버킷은 영업이익 계산에서 제외

# ✅ 감가상각 (÷12 월별 균등 배분)
annual_dep = get_annual_dep(year)   # tax_journal 우선, 없으면 직원분개장 12월 자동추출
monthly_dep = annual_dep / 12

# ✅ 손익 구조
영업이익 = 매출액 - 원재료순 - 부재료매입 - 운영비(5xx+8xx 제외버킷) - monthly_dep
이자비용 = df[df['계정코드']=='931']['차변'].sum()
영업외수익_반복  = df[df['계정코드']=='901']['대변'].sum()  # 이자수익
영업외수익_일회성 = df[df['계정코드'].isin(['914','923','924'])]['대변'].sum()
실질이익 = 영업이익 + 영업외수익 - 이자비용

## 10. 비용 대분류 cost_bucket (확정 — 2026-06-04 전면 개편)

```python
def cost_bucket(code, name, note="") -> Optional[str]:
    # ── 원재료·부재료 ─────────────────────────────────────────────────────
    if c==153 or "원재료" in n: return "원재료"
    if c==162 or "부재료" in n: return "부재료"

    # ── 환경 (오니/슬러지 위탁처리 — 적요 우선) ─────────────────────────
    if any(k in note for k in ["오니","슬러지"]): return "환경"

    # ── 인건비 (실 급여만) ────────────────────────────────────────────────
    if any(k in n for k in ["급여","임금","상여","잡급"]): return "인건비"
    if "대광용역" in n: return "인건비"
    if "경상연구개발" in n: return "인건비"  # 연구원 급여형태

    # ── 퇴직급여 ──────────────────────────────────────────────────────────
    if any(k in n for k in ["퇴직","DC형"]): return "퇴직급여"

    # ── 4대보험 ───────────────────────────────────────────────────────────
    if any(k in n for k in ["건강보험료","국민연금","고용보험","산재보험"]): return "4대보험"
    if "복리후생" in n:
        if any(k in note for k in ["건강보험","장기요양","국민연금","고용보험","산재"]):
            return "4대보험"
        return "복리후생"

    # ── 전력·수도 / 일회성손익 / 안전관리비 / 하드웨어 ─────────────────────
    if any(k in n for k in ["전력","가스수도","수도료","가스","수도"]): return "전력·수도"
    if "안전관리비" in n: return "안전관리비"
    if "회사설정계정과목" in n:
        if any(k in note for k in ["안전","마스크","소방","중대재해","크레인","방진","보호구"]):
            return "안전관리비"
        return "유지·소모품"
    if "소모공구비" in n: return "하드웨어"
    if any(k in n for k in ["유형자산처분","무형고정자산상각","감가상각비"]): return "일회성손익"
    if "감가상각누계" in n: return None  # 비용 아님
    if "대손상각" in n: return "대손상각"
    if any(k in n for k in ["운반","차량유지","차량렌트","여비교통"]): return "물류·차량"

    # ── 지급수수료 — 적요 기반 재분류 ────────────────────────────────────
    if "지급수수료" in n:
        if any(k in note for k in ["조정료","기장료","수수료","월정료","연회비","추심"]): return "수수료"
        if any(k in note for k in ["오니","슬러지","방제"]): return "환경"
        if any(k in note for k in ["소방","안전","보일러","방화"]): return "안전관리비"
        if any(k in note for k in ["하치장","중개","운반","물류"]): return "물류·차량"
        if any(k in note for k in ["시험","인증","심사","연구","검사"]): return "연구개발"
        return "수수료"

    if "카드수수료" in n: return "수수료"
    if any(k in n for k in ["임차료","임차"]): return "임차료"  # 세금과공과에서 독립
    if "세금과공과" in n: return "세금·공과"
    if any(k in n for k in ["소모품","수선"]): return "유지·소모품"
    if "보험" in n: return "보험료"
    if "연구개발" in n: return "연구개발"
    if any(k in n for k in ["접대","기업업무추진","통신","사무용품",
                             "도서인쇄","교육훈련","광고","잡비"]): return "판관비경비"
    return "기타"
```

|---|---|---|

| 복리후생 | ✅ | 식대·경조사 등 |
| 일회성손익 | ❌ | 영업이익 아래 별도 |
| 환경 | ✅ | 오니 위탁처리 |
| 임차료 | ✅ | 세금과공과에서 분리 |
| 나머지 | ✅ | 정상 영업비용 |

## 11. 매출채권 처리 원칙 (확정)

```
발생액 = 108 차변 합계 (창업일~기준월 전체 이력)
회수액 = 108 대변 합계 (어음수취 포함 — 결제 의무 이행 기준)
잔액   = 발생액 - 회수액
회수율 = 회수액 / 발생액
DSO    = FIFO 매칭 후 (어음수취일 - 발생일) 가중평균

어음수취(110) 별도 추적:
  110 잔액 = 어음 수취 후 미현금화 금액 (부도 리스크 지표)
  어음 현금화: 103 차변 + 110 대변 + 적요 '추심'

load_journal_upto(): 날짜 제한 없이 전체 이력 조회 (정확도 우선)
```

## 17. 페이지별 핵심 로직 요약

### Home.py
```
기준년도·월 선택 → 최신월 자동 선택
건강점수 도넛 + 4대 신호등 카드
손익 흐름 표: 매출→원재료→인건비→감가상각→영업이익→이자→실질이익
비용구조 도넛 + 거래처별 매출 바차트
경영 데이터 JSON 다운로드
```

### 1_매출채권관리.py
```
전체 이력 누적 로드 (날짜 제한 없음)
연령분석 (FIFO): 정상/주의/경고/악성
거래처별 결제기간 분포 차트 + 권장여신 자동 산출
110 받을어음 미현금화 잔액 표시
```

### 2_원가비용통제.py
```
월별 / 연간 토글
비용 대분류 바차트 (절대금액 / 매출대비%)
Drill-down: 대분류 → 계정별 → 전표 상세
감가상각 월배분 자동 반영
```

### 3_자금타임라인.py
```
보통예금(103) 월별 입금/지출 P&L 방식 표시
  입금: 매출채권수금·어음추심·기타
  지출: 원재료결제·부자재결제·이자이체·인건비·기타
설비투자 이력 (Code 206/208/240)
이자비용 월추이 차트
대출 만기 경보
```

### 5_데이터허브설정.py
```
탭1: 분개장 업로드 (journal 테이블)
탭2: 세무사 분개장 업로드 (tax_journal 테이블)
     → Code 518/818/840 자동 추출 → ÷12 미리보기
탭3: 블랙리스트
탭4: 대출 정보
```

### 6_연간비교.py
```
연도 복수 선택 (최대 4개)
손익비교표: 항목별 금액(%)+전년대비증감
  행: 매출→원재료→부재료→인건비→퇴직급여→4대보험→복리후생→...→감가상각→영업이익→이자→실질이익
비용구조 스택 바차트
인건비율·원재료율 추세 라인차트
감가상각 자동 반영 (손익계산서와 동일 기준)
```

### 7_손익계산서.py
```
월별 / 연간 토글
감가상각 ÷12 자동 배분 표시
세부 비용 항목 (인건비/퇴직급여/4대보험/복리후생/임차료 분리)
영업외수익·이자비용·일회성손익 별도 라인
인쇄/PDF 저장
```
