## 1. 프로젝트 정의

**한 줄 정의:** 단순 CFO 대시보드가 아닌, 회사의 방향성을 결정하는 경영 의사결정 지원 도구.
인건비 비중이 높아 줄여야 하는지, 원재료가 상승하는지 등 모든 방향성을 판단할 수 있는 툴.
CEO / CFO 최고관리자 단 1인 전용.

**세 가지 역할:**
1. 과거 진단 — 숫자가 어떻게 변했나 (추세)
2. 현재 신호 — 지금 어디가 문제인가 (경보)
3. 방향 근거 — 이 추세면 다음 달 어떻게 되나 (판단 근거)

**사용자:** 경영자 단 1인. 차변/대변 화면 노출 금지.

**입력:** 위하고 월별 분개장 CSV.
- 보유 데이터: 2022년(4~12월) / 2023 / 2024 / 2025 전체
- 매달 1개월치 추가 업로드로 누적
- 2022년은 공장 매매 이벤트 있는 해 (자산 급변 정상)

**경영 데이터 내보내기:**

```python
def load_and_clean(filepath):
    try: df = pd.read_csv(filepath, encoding='utf-8')
    except: df = pd.read_csv(filepath, encoding='cp949')
    df = df[df['구분'].isin(['차변','대변'])].copy()
    df['전표일자'] = pd.to_datetime(df['전표일자'], errors='coerce')
    df = df[df['전표일자'].notna()]
    for col in ['차변','대변']:
        df[col] = pd.to_numeric(
            df[col].astype(str).str.replace(',',''), errors='coerce').fillna(0)
    df['그룹'] = df['Code'] // 100
    return df
```

## 10. 매출채권(AR) FIFO 로직 (재작성 필요)

**현재 문제:** aging.py 전표번호 기반 광범위 매칭 → 모든 거래처 악성으로 뜸

**올바른 로직:**

```python
def extract_ar_collections(df):
    collections = []
    for vno, grp in df.groupby('전표번호'):
        ar_credits = grp[(grp['Code']==108) & (grp['대변']>0)]
        for _, ar in ar_credits.iterrows():
            거래처 = ar['거래처']
            금액 = ar['대변']
            적요 = str(ar['적요'])
            수단 = '현금'
            if '받을어음' in 적요 or '어음' in 적요 or '보관' in 적요:
                수단 = '어음수취'
            bill_in = grp[(grp['Code']==110) & (grp['차변']>0)]
            if len(bill_in) > 0 and 수단=='현금':
                if 거래처 in bill_in['거래처'].values:
                    수단 = '어음수취'
            collections.append({
                '전표번호': vno, '일자': ar['전표일자'],
                '거래처': 거래처, '회수액': 금액, '수단': 수단
            })
    return pd.DataFrame(collections)
```

**예외 케이스:**
- 103 차변 + 108 대변 → 현금 회수 (거래처는 108 대변 기준)
- 101 차변 + 108 대변 → 현금 회수
- 110 차변 + 108 대변 → 어음수취 (미현금)
- 103 차변 + 110 대변 + 적요 '받을어음추심' → 어음현금화
- 960(잡손실) 절사: 108 대변 금액 기준 처리
- FIFO: 거래처별 108 차변 날짜 오래된 것부터 차감
- DSO = 발생일~회수일 날짜차이 거래처별 가중평균

**AR 핵심 사항:**
- AR 발생 = 매출액 × 1.1 (부가세 포함) → "매출채권은 부가세 포함" 안내
- 매입채무(251) 추적 불필요 — 잔액만 확인
- 2026년 카드 입금 케이스 추가 필요 (실제 전표 확인 후)

## 12. 비용 대분류 cost_bucket (미분류 0건 검증 완료)

```python
def cost_bucket(code, name, note: str = "") -> Optional[str]:
    n = str(name); note = str(note)
    try:
        c = int(float(str(code))) if str(code).replace(".", "").isdigit() else 0
    except: c = 0

    if c == 153 or any(k in n for k in ["원재료비","원재료"]): return "원재료"
    if c == 162 or any(k in n for k in ["부재료비","부재료"]): return "부재료"
    if any(k in n for k in ["급여","임금","상여","퇴직","잡급","복리후생","DC형"]): return "인건비"
    if "대광용역" in n: return "인건비"
    if any(k in n for k in ["전력","가스수도","수도료","가스","수도"]): return "전력·수도"
    if "안전관리비" in n: return "안전관리비"
    if "회사설정계정과목" in n:
        if any(k in note for k in ["안전","마스크","소방","중대재해","크레인","방진","보호구"]):
            return "안전관리비"
        return "유지·소모품"
    if "소모공구비" in n: return "하드웨어"  # 손익 포함(기본) + 토글로 제외 가능
    if any(k in n for k in ["유형자산처분","무형고정자산상각","감가상각비"]): return "일회성손익"
    if "감가상각누계" in n: return None  # 상계항목 — 비용 아님
    if "대손상각" in n: return "대손상각"
    if any(k in n for k in ["운반","차량유지","차량렌트","여비교통"]): return "물류·차량"
    if "지급수수료" in n:
        if any(k in note for k in ["조정료","기장료","수수료","월정료","연회비","추심"]): return "수수료"
        if any(k in note for k in ["오니","슬러지","방제","소방","안전","검사"]): return "안전관리비"
        if any(k in note for k in ["하치장","중개","운반","물류"]): return "물류·차량"
        if any(k in note for k in ["시험","인증","심사","연구"]): return "연구개발"
        return "수수료"
    if "카드수수료" in n: return "수수료"
    if any(k in n for k in ["세금과공과","임차료","임차"]): return "세금·임차"
    if any(k in n for k in ["소모품","수선"]): return "유지·소모품"
    if "보험" in n: return "보험료"
    if "연구개발" in n: return "연구개발"
    if any(k in n for k in ["접대","기업업무추진","통신","사무용품",
                             "도서인쇄","교육훈련","광고","잡비"]): return "판관비경비"
    return "기타"
    # ⚠️ "기타" 발생 시 대시보드에 "미분류 항목 발생" 경보 표시
    # → Claude Code에서 cost_bucket 키워드 추가
```

**⚠️ 하드웨어(소모공구비) 처리:**

## 17. UI 설계 원칙 (확정)

**기준월 선택:**
```
- 앱 실행 시 최신 데이터월 자동 선택
- 기준년도 드롭다운 + 월 버튼 분리
- 적재 안 된 월 비활성화
- "현재 기준: 2025년 12월" + "최신 데이터" 초록 태그
- 년도 변경 시 해당 연도 마지막 적재월 자동 선택
```

**홈 화면 레이아웃:**
```
[기준년도 + 기준월 선택기]

[건강점수 도넛] + [4대 신호등 카드]
  영업이익률 / 인건비비율 / 매출채권회수율 / 원재료매입

[이번 달 손익 테이블]
  매출액 100%
  매입/비용합계 → 하위항목 계층구조 (모든 % = 매출대비)
  영업이익 → 영업외수익 → 이자비용 → 일회성손익 → 실질이익

[비용구조 도넛]

[매출채권 현황 테이블]
  거래처 / 미수금 / 회수율 / 경과일 / 위험도 뱃지

[거래처 집중도 바차트]
  상위 거래처 매출 비중 + 집중도 경보

[경영 데이터 내보내기 JSON 버튼]
```

**디자인 원칙:**
- 카드 기반, 도넛 차트, 바차트 구조
- 위험/주의/정상 뱃지 자동 표시
- 경고 자동으로 빨갛게 + 사유 한 줄
- 회계용어(차변/대변) 절대 금지
- plotly 우선, 모바일 대응
- 미분류(기타) 항목 발생 시 경보 표시
