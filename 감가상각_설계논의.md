# 감가상각비 처리 설계 논의

## 1. 현재 상황

### 프로젝트 개요
- 곡성안전유리 CFO 대시보드 (Streamlit + Supabase)
- 입력: 위하고 월별 분개장 CSV (직원이 직접 입력)
- 목적: CEO/CFO 단 1인이 보는 월간 경영 의사결정 도구 (재무제표 아님)

### 분개장 구조 (위하고 특성)
- 직원 분개장: 매월 업로드 (매출, 매입, 인건비 등)
- 세무사 분개장: 별도 존재, 동일한 위하고 형식

### 데이터 확인 결과

**Code 518 감가상각비(제) — 연도별 현황:**
- 2022~2024: ❌ 직원 분개장에 감가상각비 항목 없음
- 2025-09: Code 518 감가상각비(제) 6,721,875원 (구형 세척기 매각 시 잔여상각)
- 2025-12: Code 518 감가상각비(제) 709,774,356원 / Code 818 40,609,635원 / Code 840 36,061,000원
  - 합계 786,444,991원 (적요: "당기분 감가상각비 설정")

**결론:** 세무사무소 분개장의 연말 감가상각 설정이 2025년부터 직원 분개장에도 합산된 것으로 추정.
2022~2024년은 세무사 분개장에서만 처리되어 직원 분개장에 없음.

---

## 2. 핵심 문제

### 문제 1: 연도간 비교 왜곡
```
2022~2024 영업이익 = 감가상각 없이 계산됨 → 실제보다 높음
2025     영업이익 = 감가상각 포함됨       → 상대적으로 낮음
```
같은 기준이 아니라 연간 비교가 의미 없어짐.

### 문제 2: 12월 집중 현상
- 1~11월: 감가상각 = 0
- 12월: 786백만원 일괄 계상
→ 12월 손익이 왜곡됨

### 문제 3: 이중계상 우려 (해소됨)
- 기계 구매 → Code 206 기계장치 (자산, 2xx) → 손익 필터(5xx/8xx)에 안 잡힘
- 감가상각비 → Code 518 (5xx) → 손익에 반영
- **이중계상 없음. 구매는 자산으로만 처리됨.**

## 3. 현재 임시 처리 방식 (Claude Code가 적용한 것)

감가상각비(제)를 "자산처분" 버킷으로 분류 → 영업이익 아래로 분리

```python
# metrics.py
EXCLUDE_FROM_영업 = ["원재료", "부재료", "자산처분"]
운영비_기타df = cost_df[~cost_df["대분류"].isin(EXCLUDE_FROM_영업)]
영업이익_v7 = 매출액 - 원재료순 - 부재료매입 - 운영비_기타
실질이익 = 영업이익_v7 - 이자비용 - 자산처분손실
```

**이 방식의 문제:**
- 영업이익이 감가상각 없이 계산되어 실제보다 높게 표시됨
- "자산처분손실"이라는 잘못된 라벨로 표시됨 (실제는 감가상각)
- 개념적으로 올바르지 않음 (감가상각은 영업이익 안에 있어야 함)

## 4. 검토할 옵션

### 옵션 A: 연간 감가상각을 12등분해서 월별 반영
- 세무사에게 연간 감가상각 스케줄 받음 (건물/기계/차량/무형자산별)
- Supabase `depreciation` 테이블에 연도별 입력
- 대시보드가 ÷12 해서 매월 균등 배분
- 2022~2024 소급 적용 가능
- **장점:** 월별 손익이 정확하고 일관됨
- **단점:** 수동 입력 필요 (연 1회), 세무사와 협의 필요

### 옵션 B: 현재 방식 유지 + 라벨만 수정
- "자산처분손실" → "감가상각비(연말설정)"로 이름 변경
- 영업이익 아래 표시 유지
- 2022~2024는 0으로 표시
- **장점:** 구현 간단
- **단점:** 연도간 비교 왜곡 지속, 개념적으로 부정확

### 옵션 C: 세무사 분개장 CSV 별도 업로드
- 감가상각비 행만 필터링해서 `depreciation` 테이블에 저장
- 위하고 동일 형식이라 파싱 어렵지 않음
- 매년 세무사 결산 후 업로드
- **장점:** 가장 정확, 수작업 최소
- **단점:** 업로드 프로세스 추가 필요
